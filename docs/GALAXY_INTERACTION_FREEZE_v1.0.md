# 图谱交互冻结文档 v1.0

本文档冻结“雅努斯词境 OS / TechLex OS”当前图谱页面的人机交互、视觉效果、设置顺序和验收标准。

目标是：即使未来从 0 重新开发，只要遵守本文档，也能复刻当前已经确认满意的图谱体验。

本文档是图谱体验的产品级冻结规格，不是宪法正文，也不是一次性开发计划。未来任何图谱交互改动，都必须先对照本文档判断是否破坏已冻结体验。

## 1. 上层定位

图谱不是装饰功能，也不是为了展示复杂关系而展示复杂关系。

图谱只回答一个核心问题：

```text
这个词为什么记不住，应从哪个场景、关系或来源重新想起？
```

图谱的价值顺序：

1. 帮用户看清词与场景、来源、标签、词族、易混词之间的关系。
2. 帮用户诊断记忆困难的来源。
3. 帮用户从关系网络重新理解单词，而不是孤立背单词。
4. 保持类似 Obsidian 全局图谱的低噪声、灰阶、可拖拽、可缩放、可检查体验。

## 2. 页面结构冻结

底栏点击 `图谱` 后，页面只展示图谱本体和极少必要入口。

页面结构：

```text
左上：图谱图标
中上：关系图谱标题
右上：设置齿轮
右上第二行：柔和的笔杆/铅笔图标
主体：白底关系图谱 Canvas
```

### 2.1 左上图谱图标

左上图谱图标不是普通装饰图标。

它同时承担：

```text
刷新图谱
清除当前高亮节点
关闭当前选中状态
重新整理布局
恢复未选中状态
```

不得再额外增加独立刷新按钮，除非后续体验证明左上图标承载不够清晰。

### 2.2 中上标题

标题固定为：

```text
关系图谱
```

PC 端标题不能与图谱节点重叠。图谱自适应视图必须预留顶部安全空间。

### 2.3 右上齿轮

齿轮只打开：

```text
关系诊断
```

它用于筛选和决定“看什么关系”。

### 2.4 右上铅笔

铅笔只打开：

```text
显示微调
```

它用于调整图谱呈现方式，不承担筛选含义。

## 3. 关系诊断设置冻结

关系诊断面板采用紧凑右侧浮层，不进入新页面。

浮层顶部包含：

```text
标题：关系诊断
重置按钮
关闭按钮
```

核心说明文案：

```text
图谱只回答一个问题：这个词为什么记不住，应从哪个场景、关系或来源重新想起。
```

说明下面是搜索框：

```text
查词 / 来源 / 标签
```

### 3.1 设置顺序

关系诊断的设置顺序必须保持宏观到微观：

```text
词频
记忆状态
场景范围
关系焦点
```

不能再改回“关系焦点优先”，因为那会让用户先选抽象诊断口径，再选基础范围，认知顺序更重。

### 3.2 折叠规则

四个设置项都必须是可折叠的。

规则：

```text
点击一个设置项，该项展开。
切换到另一个设置项，上一个自动折叠。
再次点击当前展开项，该项折叠。
允许所有设置项都处于折叠状态。
```

也就是“单开 + 可全关”。

不得强制始终有一个设置项展开。

### 3.3 选中值展示

每个设置项的当前选中值必须显示在字段标题右侧，用小胶囊展示。

目的：

```text
减少垂直空间
让用户不展开也知道当前筛选条件
避免长设置页
```

## 4. 词频设置冻结

词频选项：

```text
全部
F1高
F2中高
F3中
F4低
```

词频用于控制图谱显示哪些重要度层级的词卡。

UI 要求：

```text
按钮紧凑
选中态使用主色红
全部选项不可写成“全部词频”
手机端不允许横向溢出
```

## 5. 记忆状态设置冻结

记忆状态是用户可理解的分组，不直接暴露全部内部状态。

映射关系：

```text
全部      = 不限制
未学习    = new 或无记忆状态
学习中    = learning / reinforce / downgrade
待复习    = reviewing / due / overdue
已掌握    = release
```

注意：

```text
学习中不是复习中。
待复习才对应今日复习队列语义。
已掌握是稳定掌握或释放状态。
```

## 6. 场景范围设置冻结

场景范围用于控制图谱的学习场景边界。

要求：

```text
场景显示必须是中文。
存储 ID / tag 不强制中文化。
显示层负责把默认 domain_pack 和 scene_tag 转成中文。
```

选择逻辑：

```text
一级场景单选。
二级场景单选。
一级和二级必须级联。
全部一级场景与全部二级场景统一显示为“全部”语义。
```

手机端必须自适应，不能横向溢出。

## 7. 关系焦点设置冻结

关系焦点默认值必须是：

```text
全景
```

关系焦点是最后一层诊断镜头。

选项与含义：

```text
全景：显示全部关系，用于先看整体结构。
记忆诊断：看场景、易混、词族和来源，判断为什么记不住。
场景提取：看词属于什么真实任务场景，强化场景触发。
混淆修复：看易混词、同义词和词族，修正错误边界。
来源验证：回到来源和标签，确认这个词从哪里来、怎么用。
语义扩展：看同义词、词族、标签和双向链接，建立小网络。
```

关系焦点本质是学习诊断视角，不是普通边类型开关。

## 8. 显示微调设置冻结

显示微调由右上铅笔图标进入。

显示微调只影响视觉呈现，不改变词卡学习状态，不改变 ReviewEvent，不改变 FSRS。

当前保留项：

```text
图谱清晰度：100 / 250 / 500 张词卡
显示记忆颜色：开关
显示方向箭头：开关
节点大小：滑杆 + 数字输入
连线粗细：滑杆 + 数字输入
重新整理图谱：按钮
```

节点大小和连线粗细必须支持：

```text
拖动滑杆
手动输入数字
```

原因：手机端滑杆精细控制不稳定时，数字输入可以辅助控制。

## 9. 默认视觉风格冻结

默认图谱必须是：

```text
白底
灰阶节点
浅灰关系线
主色红只用于交互焦点
默认不铺满彩色状态
默认不显示大量文字标签
```

默认未选中节点不能太浅。手机端也必须看得见节点和整体结构。

主色红：

```text
#d11b3d
```

使用位置：

```text
选中节点
选中节点的直接连线
选中设置按钮
主操作按钮
```

## 10. 图谱渲染规则冻结

渲染组件：

```text
src/components/InteractiveGalaxyGraph.tsx
```

页面容器：

```text
src/views/GalaxyView.tsx
```

数据构建：

```text
src/wordGalaxy.ts
```

技术方式：

```text
Canvas 渲染
d3-force 力导向布局
动态 import 图谱运行时
```

图谱尺寸和缩放必须自适应：

```text
PC 端预留左侧导航和顶部标题空间。
手机端预留顶部标题和底部导航空间。
使用 100dvh，并保留 100vh fallback。
```

图谱密度档位：

```text
compact：节点数 <= 32
medium：节点数 <= 120
large：节点数 > 120
```

不同密度下，节点大小、边透明度和默认缩放可以不同，但整体必须保持清爽。

## 11. 节点交互状态机冻结

图谱节点必须采用“两步打开”规则。

### 11.1 无节点高亮时

PC：

```text
鼠标移动到节点上，可以临时点亮该节点和邻接关系。
鼠标离开后，如果没有锁定高亮，则恢复默认灰阶。
```

手机：

```text
手指触碰或滑过节点，可以临时点亮。
```

### 11.2 第一次点击/触摸节点

第一次点击或触摸某个节点时：

```text
只锁定高亮该节点。
不打开词卡详情。
直接相连的边变红。
直接邻接节点变深。
无关节点变浅。
该节点和一圈直接相连节点产生轻微动感。
```

这个动感是关系网络“被点亮”的反馈，不是装饰动画。

要求：

```text
动感轻微。
持续时间短。
不造成图谱大幅散开。
不影响后续拖拽、缩放、点击。
```

### 11.3 高亮锁定后

一旦某个节点已经被第一次点击锁定高亮：

```text
移动鼠标不能再点亮其他节点。
手机手指移动也不能随意切换高亮。
当前节点必须保持高亮。
```

如果要切换高亮节点，用户必须点击另一个节点。

### 11.4 第二次点击/触摸同一节点

第二次点击或触摸同一高亮节点：

```text
打开该节点详情卡片。
```

如果详情已经打开，再次点击同一节点：

```text
关闭详情并取消选中。
```

### 11.5 点击其他节点

当 A 节点已经高亮，点击 B 节点：

```text
B 节点成为新的高亮节点。
不立即打开 B 的详情。
再次点击 B 才打开详情。
```

这样可以避免用户只是切换观察目标时误弹详情。

## 12. 拖拽与缩放冻结

### 12.1 拖拽节点

按住节点并移动超过阈值后，进入拖拽。

规则：

```text
拖拽节点不会打开详情。
被拖节点不是硬编码瞬移，而是通过力导向目标牵引。
直接邻接节点会被带动。
释放后保留短暂惯性。
布局逐渐阻尼回稳。
```

拖拽必须有“拉动局部关系网络”的感觉。

### 12.2 拖动画布

在空白区域按住拖动：

```text
平移整个图谱视图。
不打开详情。
不改变学习数据。
```

### 12.3 缩放

PC：

```text
鼠标滚轮缩放。
缩放中心为鼠标位置。
```

手机：

```text
双指按住缩放。
缩放中心为双指中心。
```

缩放后，详情卡片位置要跟随当前视图重新计算。

## 13. 详情卡片冻结

详情卡片必须根据节点位置动态出现，不能固定在一个位置挡住节点。

定位原则：

```text
优先出现在节点旁边。
节点在左侧时，详情向右展开。
节点在右侧时，详情向左展开。
节点在上方时，详情向下展开。
节点在下方时，详情向上展开。
必须避开顶部标题区和右侧设置按钮区。
手机端不能超出屏幕。
```

关闭方式：

```text
点击详情卡片右上角 X。
再次点击已打开详情的同一节点。
点击左上图谱图标刷新清空。
```

### 13.1 词卡节点详情字段

词卡节点详情至少展示：

```text
词卡 · 记忆状态 · 词频
英文单词
词性
音标
中文释义或英文释义摘要
场景
来源
关系线索
```

关系线索包括：

```text
易混
同义
词族
标签
链接
```

这些字段服务于图谱核心价值：让用户知道这个词应该从哪个场景、来源或关系重新想起。

### 13.2 非词卡节点详情字段

非词卡节点包括：

```text
一级场景
二级场景
来源
同义关系
易混关系
词族
标签
双向链接
```

详情至少展示：

```text
节点类型
关联单词数量
节点名称
诊断提示
相关单词
```

## 14. 左上刷新规则冻结

点击左上图谱图标必须执行：

```text
清除 selectedNodeId
清除 highlightedNodeId
清除 hoveredNodeId
清除 activeNodeId
关闭详情卡片
重置用户手动视图状态
重新整理布局
触发图谱重新适配视图
```

它的用户含义是：

```text
回到未选中、干净、可重新观察的图谱。
```

## 15. 空数据状态冻结

如果当前没有可渲染词卡，图谱页不应白屏。

必须显示：

```text
当前没有可渲染词卡
导入演示包后，可以预览关系图谱、拖动节点和查看详情。
导入图谱演示包
```

演示包只能用于预览 UI，不应伪装成真实用户学习数据。

## 16. 与学习数据的边界

图谱页面是关系诊断和可视化检查页面。

图谱交互不得直接写入：

```text
ReviewEvent
UserMemoryState
FSRS 调度结果
今日待复习队列
```

图谱筛选只影响图谱显示范围，不影响真实学习记录。

点击图谱节点查看详情，也不等于“用户不认识该词”。

“点击单词列表词卡 = 不认识 = browser_detail + again = 加入今日待复习”是单词列表浏览规则，不是图谱节点点击规则。

## 17. PC 端验收清单

PC 端必须满足：

```text
打开图谱后，标题不与节点重叠。
默认灰阶节点和边可见。
鼠标移动到节点上，在无锁定高亮时可临时点亮。
第一次点击节点只高亮，不打开详情。
第一次点击后，移动鼠标不会切换到其他节点高亮。
高亮节点及其邻接节点有轻微动感。
第二次点击同一节点打开详情。
点击 X 关闭详情。
再次点击同一已选节点可关闭详情。
点击另一个节点只切换高亮，不立即弹详情。
拖拽节点会带动邻接节点。
拖拽空白区域平移画布。
鼠标滚轮可缩放。
左上图谱图标可清空选中并重新整理布局。
齿轮打开关系诊断。
铅笔打开显示微调。
```

## 18. 手机端验收清单

手机端必须满足：

```text
图谱能在手机浏览器打开。
默认图谱不是 0 nodes / 0 edges。
默认节点不能淡到看不见。
第一次触摸节点只高亮，不打开详情。
第二次触摸同一节点打开详情。
高亮后手指滑动不会随意切换高亮节点。
单指拖拽节点可带动邻接节点。
单指拖动空白区域可平移画布。
双指可缩放图谱。
详情卡片不超出屏幕。
详情卡片可通过 X 关闭。
底部导航不遮挡核心操作。
关系诊断和显示微调浮层不横向溢出。
```

## 19. 不得回退的交互

以下行为不得重新引入：

```text
点击节点第一次就直接弹详情。
已高亮节点后，鼠标移动又点亮其他节点。
节点高亮后无法恢复未选中状态。
只拖动画布，不能拖动节点。
只能拖动当前节点，邻接节点完全不动。
手机端不能双指缩放。
详情卡片固定位置，遮住被点击节点。
右上角堆叠过多按钮遮挡图谱。
默认节点过浅，手机端几乎看不见。
图谱筛选页过长，所有设置项同时展开。
关系焦点默认不是“全景”。
场景分类显示英文 ID。
图谱点击行为误写入复习数据。
```

## 20. 代码位置索引

当前实现入口：

```text
页面入口：src/views/GalaxyView.tsx
图谱渲染与交互：src/components/InteractiveGalaxyGraph.tsx
图谱数据构建：src/wordGalaxy.ts
底栏入口：src/App.tsx
```

关键状态：

```text
selectedNodeId：已打开详情的节点
highlightedNodeId：第一次点击后锁定高亮的节点
hoveredNodeId：无锁定状态下的临时 hover 节点
activeNodeId：正在 pointer down / drag 的节点
animationSignal：外部触发重新整理布局
transform：当前平移和缩放
```

关键动作：

```text
renderGraph：绘制节点、边、标签、高亮和弱化状态
pulseNodeNeighborhood：首次点亮节点时给邻接节点轻微动感
startGentleMotion：首次进入图谱时的轻微首屏游动
applyFitView：适配视图
resetLayout：清除高亮并重排
computeDetailPosition：详情卡片动态定位
handlePointerDown / Move / Up：PC 和手机统一 pointer 交互
handleWheel：PC 滚轮缩放
startPinchFromActivePointers / updatePinchViewport：手机双指缩放
```

## 22. 冻结结论

当前图谱体验的核心不是“更多功能”，而是：

```text
少入口
低噪声
灰阶全局
红色焦点
两步打开
邻接动感
可拖拽
可缩放
可关闭
可诊断
不污染学习数据
```

未来所有图谱改造，都必须先证明不会破坏以上冻结体验。
