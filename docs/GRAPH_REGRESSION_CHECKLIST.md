# Graph Regression Checklist

本文档是每次修改图谱后必须执行的回归清单。它用于避免重复出现“拖不动、太花、不能恢复灰阶、手机端不可用”等问题。

## 1. 修改前检查

修改图谱代码前，先确认已阅读：

```text
docs/GRAPH_EXPERIENCE_CONSTITUTION.md
docs/OBSIDIAN_GRAPH_REFERENCE_SPEC.md
docs/adr/ADR-graph-renderer.md
```

确认本次修改属于哪一类：

```text
视觉样式
图谱物理
点击详情
移动端触控
性能优化
数据关系
图谱筛选
```

如果修改会改变默认图谱体验，必须先说明理由。

## 2. 自动化检查

每次图谱改动后运行：

```powershell
corepack pnpm run typecheck
corepack pnpm run build
corepack pnpm run smoke:phase10
```

通过标准：

```text
typecheck 无错误
build 无错误
smoke:phase10 通过
浏览器控制台无新增 error
```

Webpack 性能 warning 单独评估，不能掩盖功能回归。

## 3. PC 默认视图检查

打开图谱页后检查：

```text
[ ] 图谱不是空白
[ ] nodes > 0
[ ] edges > 0
[ ] 默认是灰阶低噪声
[ ] 默认没有彩色环形图
[ ] 默认没有密集文字标签
[ ] 默认视图能看到整体结构
[ ] 节点和边不过黑、不过粗、不过密
[ ] “适合视图”按钮有效
[ ] “重新布局”按钮有效
```

失败即阻断：

```text
默认显示 0 nodes / 0 edges
默认图谱过小看不清
默认图谱太花
默认出现彩色噪声
```

## 4. PC 点击详情检查

点击一个词卡节点：

```text
[ ] 出现详情面板或浮层
[ ] 直接显示英语词
[ ] 能看到中文或核心解释
[ ] 能看到相关词 / 标签 / 场景 / 来源中的至少一类
[ ] 详情可以关闭
[ ] 关闭详情后图谱仍可操作
```

点击 Hub / 标签 / 场景 / 来源节点：

```text
[ ] 显示节点类型
[ ] 显示关联词数量
[ ] 显示关联词列表或 chips
[ ] 详情可以关闭
```

失败即阻断：

```text
点击节点没有内容
详情不能关闭
详情遮挡图谱且无法恢复
```

## 5. PC 拖动检查

拖动一个节点：

```text
[ ] 节点能被拖动
[ ] 焦点节点临时高亮
[ ] 直接边临时增强
[ ] 直接邻居被牵引
[ ] 背景节点淡化但仍可见
[ ] 拖动过程平滑
[ ] 没有硬跳动
[ ] 没有全图失控乱飞
```

释放节点：

```text
[ ] 节点有短暂自然回弹
[ ] 运动逐步衰减
[ ] 红色高亮恢复为灰阶
[ ] 详情可保留
[ ] 视口不被强行重置
```

失败即阻断：

```text
一点都拖不动
只拖当前节点，邻居完全不动
释放后红色不恢复
释放后突然跳回
释放后图谱乱飞
```

## 6. PC 缩放和平移检查

```text
[ ] 鼠标滚轮可缩放
[ ] 拖动画布可平移
[ ] 缩放后点击节点仍准确
[ ] 平移后点击节点仍准确
[ ] 用户手动缩放后不会被自动 fit 抢回
```

## 7. 手机端检查

使用手机或移动端模拟器检查：

```text
[ ] 图谱页不是空白
[ ] nodes > 0
[ ] edges > 0
[ ] 默认视图足够大
[ ] 单指可拖动画布
[ ] 单指可拖动节点
[ ] 拖动节点时邻居跟随
[ ] 释放后自然回弹
[ ] 双指可缩放
[ ] 点击节点可打开详情
[ ] 详情可关闭
[ ] 底部导航不遮挡图谱核心操作
```

失败即阻断：

```text
手机显示 0 nodes / 0 edges
手机图谱无法拖动
手机不能双指缩放
手机点击节点没有详情
```

## 8. 视觉噪声检查

默认图谱必须避免：

```text
[ ] 黑点过多
[ ] 彩色过多
[ ] 标签文字过多
[ ] 边线过粗
[ ] Hub 节点过大
[ ] 背景太暗
[ ] 选中态残留
```

通过标准：

```text
默认看全局
交互看邻域
详情看内容
颜色只在需要时出现
```

## 9. 数据关系检查

导入 Obsidian-grade 测试包后检查：

```text
[ ] 词卡节点存在
[ ] 场景节点存在
[ ] 来源节点存在
[ ] 标签节点存在
[ ] 同义词节点存在
[ ] 易混词节点存在
[ ] 词族节点存在
[ ] 掌握状态节点存在
[ ] links_to / backlinks 可见
```

如果新增词卡字段，应检查是否影响图谱关系生成。

## 10. 性能检查

大包检查：

```text
[ ] 450 cards 可打开图谱
[ ] 2000 cards 不应卡死
[ ] 渲染上限生效
[ ] 页面不会因图谱导致主线程长时间无响应
[ ] 手机端可退化但不可空白
```

性能优化不得删除核心交互：

```text
点击详情
拖动节点
邻居牵引
释放回弹
缩放
关闭详情
```

## 11. 回归记录模板

每次图谱改动后，在阶段报告或 `PROJECT_STATE.md` 中记录：

```text
Graph regression:
- typecheck:
- build:
- smoke:phase10:
- PC default view:
- PC click detail:
- PC drag and release:
- mobile default view:
- mobile pinch zoom:
- known issue:
- screenshot/video evidence:
```

## 12. 不得通过的情况

出现以下任一情况，不得宣布图谱阶段通过：

```text
图谱空白
手机端 0 nodes / 0 edges
节点无法拖动
双指缩放不可用
点击节点无法看词
详情无法关闭
释放后颜色不恢复
默认视图过花
用彩色装饰图替代灰阶知识图谱
图谱改动污染 Domain Core
```
