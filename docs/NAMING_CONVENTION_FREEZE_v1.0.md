# 命名体系冻结文档 v1.0

本文档冻结“雅努斯词境 OS”的产品命名体系。

目标是避免未来把底座系统、场景词汇网络、历史开发代号混在一起，导致品牌、文档、代码、词卡包和 UI 表述漂移。

## 1. 命名总原则

本项目分两层命名：

```text
底座系统名：雅努斯词境 OS
场景应用名：雅努斯词境 OS · {场景名}词汇网络
```

底座系统代表通用学习运行时。

场景应用代表在某个具体学习目标下配置出来的词汇网络。

## 2. 底座系统名

冻结名称：

```text
雅努斯词境 OS
```

含义：

```text
个人场景英语词汇学习操作系统
自动学习运行时
可导入词卡包、运行 FSRS、记录 ReviewEvent、更新 UserMemoryState、展示图谱关系的通用底座
```

底座系统不绑定任何单一场景。

不得写死：

```text
技术英语
AI 编程英语
Web3
中考英语
高考英语
任何具体用户或具体词卡包
```

## 3. 场景应用命名公式

冻结公式：

```text
雅努斯词境 OS · {场景名}词汇网络
```

示例：

```text
雅努斯词境 OS · 技术英语词汇网络
雅努斯词境 OS · 初中英语词汇网络
雅努斯词境 OS · 高中英语词汇网络
雅努斯词境 OS · 中考英语词汇网络
雅努斯词境 OS · 高考英语词汇网络
雅努斯词境 OS · 考研英语词汇网络
雅努斯词境 OS · 职场英语词汇网络
雅努斯词境 OS · 外贸英语词汇网络
雅努斯词境 OS · 医学英语词汇网络
雅努斯词境 OS · 法律英语词汇网络
```

短名称可用于 UI 局部区域：

```text
技术英语词汇网络
技术词汇网络
高中英语词汇网络
初中英语词汇网络
```

但正式文档、产品介绍、导出包说明中应保留完整命名：

```text
雅努斯词境 OS · 技术英语词汇网络
```

## 4. 当前场景应用

当前正在开发和验收的场景应用冻结为：

```text
雅努斯词境 OS · 技术英语词汇网络
```

短别名：

```text
技术英语词汇网络
技术词汇网络
```

历史开发代号：

```text
TechLex OS
```

说明：

```text
TechLex OS 可继续作为技术英语词汇网络的历史代号、代码代号或旧文档检索关键词。
TechLex OS 不再作为最终对外主品牌。
TechLex OS 不代表雅努斯词境 OS 的全部。
```

## 5. 技术英语词汇网络内部场景

`技术英语词汇网络` 是当前场景应用。

它内部可以继续包含一级场景和二级场景。

示例：

```text
一级场景：AI编程
二级场景：IDE界面、AI代码助手、依赖管理、命令行终端

一级场景：Web3
二级场景：钱包、智能合约、区块浏览器、链上交易

一级场景：云与协作
二级场景：部署、权限、日志、工作区
```

这些场景是词卡组织结构，不是产品主品牌。

## 6. 不同词汇网络共享的底座能力

所有词汇网络共享：

```text
标准词卡包导入
词卡生产标准
学习计划
FSRS 记忆调度
ReviewEvent
UserMemoryState
今日待复习
单词本
统计
图谱
设置
本地存储与备份
UI 组件系统
```

不同词汇网络只替换：

```text
DomainPack
一级场景
二级场景
词卡包
来源标准
词频规则
学习目标
例句语境
图谱关系密度
```

不得为每个词汇网络复制一套学习内核。

## 7. 对外展示规则

产品标题：

```text
雅努斯词境 OS
```

当前应用或当前词卡包上下文：

```text
技术英语词汇网络
```

完整场景应用标题：

```text
雅努斯词境 OS · 技术英语词汇网络
```

浏览器标题和 PWA 名称：

```text
优先显示：雅努斯词境 OS
在场景应用独立发布时可显示：雅努斯词境 OS · 技术英语词汇网络
```

## 8. 文档命名规则

新文档优先使用底座产品名：

```text
雅努斯词境 OS
```

当文档只描述当前技术英语样板时，写法为：

```text
雅努斯词境 OS · 技术英语词汇网络
历史代号：TechLex OS
```

旧文档中的 `TechLex OS` 可以保留，不必机械重命名。

但新文档不得继续把 `TechLex OS` 写成通用产品名。

## 9. 代码和数据命名规则

代码层可以暂时保留：

```text
techlex-os
TechLex
```

原因：

```text
这些属于历史工程代号，强行重命名会带来无关风险。
```

但新增面向用户的字段应优先使用：

```text
Janus Lexicon OS
janus-lexicon-os
```

场景应用 ID 建议：

```text
technical-english-network
middle-school-english-network
high-school-english-network
zhongkao-english-network
gaokao-english-network
```

词卡包 ID 建议：

```text
janus-technical-english-network-v1
janus-high-school-english-network-v1
```

## 10. 禁止回退清单

```text
把 TechLex OS 当成整个产品的最终主品牌。
把技术英语词汇网络等同于雅努斯词境 OS。
为每个词汇网络复制一套学习算法。
在通用底座里写死技术英语场景。
在通用底座里写死中考或高考场景。
把一级场景、二级场景写成产品主品牌。
新文档继续使用 TechLex OS 代表通用产品。
重新使用“某某英语版”作为正式场景应用命名。
```

## 11. 推荐标准表述

产品底座：

```text
雅努斯词境 OS 是一个个人场景英语词汇学习操作系统。
```

当前场景应用：

```text
当前应用是“雅努斯词境 OS · 技术英语词汇网络”，历史开发代号为 TechLex OS。
```

扩展场景：

```text
未来可基于同一底座扩展“雅努斯词境 OS · 高中英语词汇网络”“雅努斯词境 OS · 中考英语词汇网络”等场景应用。
```

## 12. 冻结结论

最终命名体系冻结为：

```text
雅努斯词境 OS
  ├─ 技术英语词汇网络
  ├─ 初中英语词汇网络
  ├─ 高中英语词汇网络
  ├─ 中考英语词汇网络
  ├─ 高考英语词汇网络
  └─ 用户自定义词汇网络
```

其中：

```text
雅努斯词境 OS = 底座系统
{场景名}词汇网络 = 场景应用
TechLex OS = 技术英语词汇网络历史开发代号
```
