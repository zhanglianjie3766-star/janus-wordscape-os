# Graph Experience Constitution

本文档沉淀“雅努斯词境 OS / TechLex OS”图谱体验的长期约束。它用于防止未来重复返工，优先级低于 `docs/TECHLEX_OS_CONSTITUTION_v1.2.md`，高于普通阶段计划、UI 临时需求和单次实现方案。

## 1. 图谱定位

图谱不是装饰动画，也不是单纯的数据可视化。

图谱的核心任务是帮助用户检查和理解：

```text
这个词和哪些词有关？
这个词属于哪些场景？
这个词来自哪些来源？
这个词有哪些同义词、易混词、词族？
这个词当前掌握状态如何？
我应该优先复习哪些关联词？
```

因此，图谱必须服务于学习判断，而不是制造视觉热闹。

## 2. 长期体验基准

图谱体验长期对标 Obsidian 全局图谱，而不是营销式炫彩图、环形装饰图或普通静态关系图。

基准关键词：

```text
灰阶
清爽
低噪声
全局感
空间结构清楚
局部关系可检查
点击可看详情
拖动有力场
平滑
有惯性
有阻尼
自然回弹
```

## 3. 默认状态

默认状态必须低噪声：

```text
背景为白色或近白色
节点以灰阶为主
边线很淡
整体结构可见
不要默认显示大量文字标签
不要默认显示彩色分类
不要默认把掌握状态颜色铺满全图
不要让大节点和黑点造成压迫感
```

默认打开图谱时，用户应先看到整体结构，而不是局部过度放大、彩色噪声或一团黑点。

## 4. 交互状态

当用户点击、按住或拖动一个节点时，图谱进入临时交互态：

```text
焦点节点变红或强高亮
直接相连的边变红或更深
直接邻居节点变深
二跳邻居可以轻微增强
无关背景节点淡化但不消失
整体空间结构仍然保留
```

交互态的目标是看清“当前节点的邻域”，不是把全图改成永久高亮。

## 5. 释放状态

用户释放鼠标或手指后：

```text
红色高亮必须恢复
全图必须回到灰阶默认状态
节点详情可以保留
图谱颜色不能永久锁死
局部节点可以继续自然回弹一小段时间
```

如果释放后颜色不能恢复，视为不合格。

## 6. 拖动手感

拖动节点时必须像力场，而不是像拖动一个孤立图标。

合格表现：

```text
被拖节点跟随指针
相邻节点被牵引
相邻边像弹簧一样拉伸
局部结构会被带动
释放后局部自然回弹
运动有阻尼，不突兀抖动
运动有惯性，不僵硬停住
```

不合格表现：

```text
完全拖不动
只拖动当前节点，邻居不动
节点被硬固定
拖动后瞬移
释放后突然跳回
释放后全图乱飞
拖动整个图谱而不是节点
用装饰性循环动画替代真实力导向运动
```

## 7. 点击详情

点击节点必须能看到内容。

卡片节点至少展示：

```text
英语
中文
场景
来源
掌握状态
相关标签
相关词
```

Hub / 标签 / 来源 / 场景节点至少展示：

```text
节点名称
节点类型
关联词数量
关联词列表
```

详情必须支持关闭。关闭详情不应破坏图谱当前视图。

## 8. 移动端要求

手机端不是桌面端的缩小版。移动端必须支持：

```text
默认视图足够大
单指拖动节点
单指拖动画布
双指缩放
点击节点打开详情
详情可关闭
底部导航不遮挡图谱核心操作
```

如果手机端显示 `0 nodes / 0 edges`、图谱过小、不能双指缩放或不能拖动节点，视为阻断问题。

## 9. 数据关系要求

图谱必须表达真实学习关系，不能只画词卡点。

必须支持的节点类型：

```text
word_card
domain_pack
scene
source
tag
synonym
confusing_word
word_family
mastery_state
wikilink
```

必须支持的关系类型：

```text
belongs_to_pack
belongs_to_scene
from_source
has_tag
has_synonym
has_confusing_word
has_word_family
has_mastery_state
links_to
backlinks
```

## 10. 性能要求

图谱必须在大词库下可用。

最低要求：

```text
450 cards: 默认全图可流畅查看
2000 cards: 必须有渲染上限、抽样或虚拟化策略
手机端: 必须优先保证交互可用，不追求一次显示所有节点
```

性能优化不得牺牲核心交互：

```text
点击节点
拖动节点
邻居牵引
释放回弹
详情查看
关闭详情
缩放和平移
```

## 11. 禁止事项

以下设计不得重新引入：

```text
彩色环形图作为默认图谱
大面积高饱和颜色
默认显示密集文字标签
黑点过多造成视觉噪声
释放后高亮颜色不恢复
只有装饰动效，没有真实力导向拖拽
节点点击后看不到词
手机端无法缩放或无法拖动
新增图谱功能时污染 Domain Core
将个人技术英语场景写死进图谱核心逻辑
```

## 12. 验收句

图谱是否达标，只看：

```text
我是否能在灰阶清爽的全局图谱中，点击或拖动一个节点，
平滑地看清它和哪些词、场景、来源、标签、易混词、词族、掌握状态有关，
并能在 PC 和手机上稳定查看详情、缩放、拖动、关闭？
```

如果不能，图谱体验未达标。
