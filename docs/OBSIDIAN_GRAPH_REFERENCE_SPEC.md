# Obsidian Graph Reference Spec

本文档把用户反复确认的 Obsidian 图谱参考效果固化为可验收规格。它不是 Obsidian 克隆需求，而是“雅努斯词境 OS / TechLex OS”图谱体验的对标说明。

## 1. 参考来源

当前参考来自用户提供的 Obsidian 图谱录屏和截图，尤其是腾讯会议录屏片段中展示的全局图谱交互效果。

参考重点：

```text
低噪声灰阶全局图谱
点击或拖动节点时的红色焦点
直接关联边和邻居增强
背景节点淡化
平滑力导向拖拽
释放后的自然回弹
点击节点查看内容
```

该参考是长期体验基准，不是一次性样式建议。

## 2. Obsidian 参考效果拆解

### 2.1 默认全局视图

Obsidian 的默认图谱给人的第一印象是：

```text
白底
灰点
淡灰边
整体结构分区明显
没有大量彩色噪声
没有默认挤满文字
中心和边缘都有空间感
```

我们的图谱必须优先获得这种“一眼看全局”的感觉。

### 2.2 选中节点

当鼠标点击或按住节点时：

```text
焦点节点变红或强高亮
直接边变红或更深
邻居节点变深
无关节点变淡
用户立即知道当前节点的局部关系
```

高亮是临时交互状态，不应永久保留。

### 2.3 拖动节点

Obsidian 的拖动不是“移动一个图标”，而是“拉动一个力场”。

表现包括：

```text
焦点节点跟随鼠标
直接邻居被牵引
边像弹簧一样拉伸
局部结构顺势移动
背景结构保持稳定
释放后局部自然回弹
```

这要求实现必须有真实的力导向模拟，而不是硬编码坐标跳动。

### 2.4 释放节点

释放节点后：

```text
节点继续短暂运动
运动逐渐衰减
布局自然稳定
红色高亮恢复为灰阶
详情可以保留
```

释放后若颜色不恢复，或者节点突然僵硬停止，均不符合参考。

### 2.5 点击查看

Obsidian 点击节点的核心价值是“进入内容”。

对应到雅努斯词境 OS：

```text
点击词卡节点 = 查看词卡详情
点击场景节点 = 查看该场景下的词
点击来源节点 = 查看该来源生成的词
点击标签节点 = 查看该标签关联的词
点击易混词 / 词族节点 = 查看相关词
```

详情必须可关闭。

## 3. 我们的适配规则

Obsidian 是知识库图谱；雅努斯词境 OS 是英语学习图谱。因此不能只照抄视觉，还要服务学习。

我们必须保留：

```text
灰阶全局视图
低噪声
力导向拖拽
局部邻域高亮
点击查看详情
全局和局部切换
```

我们需要增加：

```text
掌握状态
FSRS 到期状态
学习任务相关性
来源可信度
场景分类
中文解释
例句和翻译
```

## 4. 视觉规范

默认态：

```text
背景: #ffffff 或接近白色
普通节点: 中浅灰
Hub 节点: 稍深灰，略大
普通边: 很浅灰
文字标签: 默认隐藏，只在 hover / 选中 / 详情中出现
```

交互态：

```text
焦点节点: 红色或强高亮
直接边: 红色或粉红细线
直接邻居: 深灰
二跳邻居: 中灰
无关节点: 浅灰并降低透明度
```

禁止：

```text
默认彩虹配色
默认大面积掌握状态颜色
默认密集标签
过粗边线
过大黑色节点
环形装饰布局
```

## 5. 动作规范

拖动必须具备：

```text
link spring
node repulsion
damping
velocity
post-release settling
local neighborhood movement
```

不得出现：

```text
hard pin forever
manual coordinate jump
only current node moves
fixed background nodes
release snap back
decorative loop animation
```

## 6. PC 端验收

PC 端必须满足：

```text
鼠标滚轮缩放
拖动画布
拖动节点
邻居跟随
释放回弹
点击节点详情
详情可关闭
适合视图
重新布局
默认灰阶
临时高亮释放后恢复
```

## 7. 手机端验收

手机端必须满足：

```text
默认图谱不是 0 nodes / 0 edges
默认视图可见且足够大
单指可拖动节点
单指可拖动画布
双指可缩放
点击节点可打开详情
详情可关闭
底部导航不遮挡主要操作
```

## 8. 与 Obsidian 不同的边界

雅努斯词境 OS 不追求成为完整 Obsidian：

```text
不实现 Obsidian 插件系统
不实现完整文件 vault
不实现通用 Markdown 编辑器
不实现 Canvas 白板
不实现块级引用系统
```

雅努斯词境 OS 只对标 Obsidian 图谱中与学习有关的核心体验：

```text
关系可见
邻域可查
拖动自然
整体清爽
点击进内容
```

## 9. 对开发者的硬要求

任何图谱改动前必须先读：

```text
docs/GRAPH_EXPERIENCE_CONSTITUTION.md
docs/OBSIDIAN_GRAPH_REFERENCE_SPEC.md
docs/adr/ADR-graph-renderer.md
docs/GRAPH_REGRESSION_CHECKLIST.md
```

如果改动让图谱偏离 Obsidian-like 参考，应先提交设计理由，不得直接改代码。
