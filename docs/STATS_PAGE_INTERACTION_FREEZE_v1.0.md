# 统计页交互冻结文档 v1.0

本文档冻结“雅努斯词境 OS / TechLex OS”底栏菜单 `统计` 页的指标选择、页面结构、解释弹层、视觉密度和数据口径。

统计页不是报表堆叠页。它只保留会影响复习节奏和学习量调整的最小必要指标。

## 1. 页面定位

统计页只回答一个问题：

```text
我今天的记忆任务是否健康，下一步应该优先处理哪里？
```

统计页不承担：

```text
词卡导入
单词浏览
图谱关系检查
学习规则编辑
```

## 2. 页面结构

统计页固定为四个仪表盘：

```text
今日执行
记忆健康
阶段分布
薄弱场景 Top 3
```

不得重新加入：

```text
顶部“统计”大标题
页面说明小字
大量分裂豆腐块
每个指标的小图标
厚重内部分隔线
```

## 3. 整体视觉

每个仪表盘是一个整体卡片。

冻结规则：

```text
卡片圆角统一。
卡片边框轻。
卡片间距紧凑但不拥挤。
四个仪表盘在常见手机屏幕上应尽量一屏可见。
指标数字不应过粗、过大、过黑。
字段名和数字之间保持适度呼吸感。
```

数字视觉：

```text
使用 tabular-nums。
字体权重偏 normal / medium，不使用超粗。
颜色偏 slate，而不是纯黑。
手机端不得因百分比或两位数导致错行。
```

## 4. 仪表盘标题与问号

每个仪表盘标题居中。

问号图标紧挨标题右侧，不得拉到卡片最右边。

点击问号：

```text
弹出说明卡片。
说明卡片包含本仪表盘的核心作用。
逐个说明每个指标的统计方式和含义。
```

冻结规则：

```text
同一时间只能打开一个说明弹层。
点击另一个问号时，上一个自动关闭。
说明弹层必须有关闭 X。
说明弹层内容要简洁，直击本质。
前两个仪表盘弹层优先向下展开。
后两个仪表盘弹层优先向上展开。
手机端弹层向左偏移，避免右侧被遮住。
弹层标题和说明左对齐。
弹层左侧用竖线作为视觉起头。
```

## 5. 今日执行仪表盘

指标固定为四个：

```text
今日待复习
今日已复习
今日剩余
今日新学
```

数据口径：

```text
今日待复习 = 今日已正式复习的到期词卡数 + 当前剩余待复习词卡数。
今日已复习 = 今天完成的 daily_task 待复习词卡数，不含 browser_detail。
今日剩余 = 当前还未处理的待复习词卡数，包含手动标记不会后进入今日的词卡。
今日新学 = 今天从 new 进入正式记忆循环的词卡数。
```

关键边界：

```text
browser_detail + again 可以让词卡进入今日待复习。
browser_detail 不计入今日已复习。
```

## 6. 记忆健康仪表盘

指标固定为四个：

```text
逾期词卡
困难词卡
Again率
近期保持率
```

数据口径：

```text
逾期词卡 = due_at 早于当前时间且不是 new/release 的词卡。
困难词卡 = reinforce / downgrade 阶段词卡。
Again率 = 近 30 天 daily_task 中 rating=again 的比例。
近期保持率 = 近 30 天 daily_task 中非 Again 的比例，约等于 100% - Again率。
```

冻结规则：

```text
记忆健康只使用 daily_task 作为正式复习统计。
browser_detail 不进入 Again率和近期保持率。
```

## 7. 阶段分布仪表盘

指标固定为四个：

```text
未学习
学习中
待复习
已掌握
```

映射关系：

```text
未学习 = new 或无 memory state
学习中 = learning / reinforce / downgrade
待复习 = reviewing / due / overdue
已掌握 = release
```

冻结规则：

```text
只展示四个阶段。
不得恢复五阶段或更多阶段。
不得展示粗横向进度条。
四个指标保持同一行。
```

## 8. 薄弱场景 Top 3

薄弱场景用于回答：

```text
我最应该优先处理哪个二级场景？
```

排序依据：

```text
困难词卡数
逾期词卡数
待复习词卡数
近 30 天 Again率
```

展示内容：

```text
排名
二级场景中文名
待复习数
困难数
逾期数
Again 百分比
```

视觉冻结：

```text
列表应嵌在仪表盘整体卡片内。
内部列表不使用厚重独立边框。
优先使用浅背景行或轻分隔线。
避免底部轮廓线和列表轮廓线叠加。
```

空状态：

```text
暂无薄弱场景。导入并完成复习后，这里会显示最需要处理的场景。
```

## 9. 颜色策略

统计页颜色用于轻微提示，不用于制造彩色看板。

冻结规则：

```text
主色红用于关键执行和异常压力。
琥珀色用于提醒或剩余任务。
灰色用于中性指标。
不得给每个数字都套高饱和颜色。
```

状态色不得与 FSRS 四评分按钮的颜色体系混淆。

## 10. 数据联动边界

统计页读取数据，不写入数据。

不得在统计页执行：

```text
ReviewEvent 写入
UserMemoryState 更新
词卡导入
学习计划保存
```

统计页依赖：

```text
review_events
memory_states
learning_plan
cards
scene_tags
```

## 11. 与今日页关系

今日页完成正式复习后：

```text
今日已复习增加
今日剩余减少
Again率 / 近期保持率随 daily_task 更新
阶段分布随 state_after 更新
薄弱场景随场景统计更新
```

单词列表点击词卡后：

```text
今日待复习和今日剩余可能增加
今日已复习不增加
Again率不增加
近期保持率不变化
```

## 12. 与单词本关系

单词本场景学习和词卡列表会影响统计：

```text
内联学习评分 = daily_task，进入统计。
词卡列表点击 = browser_detail，不进入正式复习统计，但可进入今日队列。
```

## 13. 不得回退

以下行为不得重新引入：

```text
统计页顶部大标题和说明小字。
四个仪表盘间距过大导致手机端需要大量滚动。
仪表盘内部使用厚重单元格边框。
每个指标加小图标。
数字过粗、过大、纯黑。
问号距离标题过远。
多个说明弹层同时打开。
说明弹层无法关闭。
browser_detail 被纳入 Again率或今日已复习。
薄弱场景列表使用多层厚边框。
```

## 14. 回归验收清单

每次改统计页后至少检查：

```text
页面没有顶部“统计”标题。
四个仪表盘顺序正确。
每个仪表盘标题居中，问号紧邻标题。
点击任一问号只打开一个说明弹层。
说明弹层可关闭。
今日执行显示 4 个指标。
记忆健康显示 4 个指标。
阶段分布显示 4 个指标。
薄弱场景 Top 3 不出现厚重嵌套边框。
数字字体不过粗、不过黑。
手机端 360/390/412/430 宽度不横向溢出。
browser_detail 不计入今日已复习和 Again率。
```
