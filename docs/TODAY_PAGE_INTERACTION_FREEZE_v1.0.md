# 今日页交互冻结文档 v1.0

本文档冻结“雅努斯词境 OS / TechLex OS”底栏菜单 `今日` 页的页面定位、交互逻辑、队列规则、UI 结构和验收标准。

本文档是产品级冻结规格，不是实现计划。未来修改今日页前，必须先确认不会破坏本文档冻结的学习队列语义。

## 1. 页面定位

`今日` 页不是新词学习入口，也不是全部词卡浏览入口。

它只回答一个问题：

```text
今天真正需要复习的词卡还有哪些？
```

今日页的核心职责：

1. 展示今日待复习任务量。
2. 只处理已经到期或被产品规则加入今日待复习的词卡。
3. 执行正式 FSRS 评分复习。
4. 把正式复习写入 `review_mode=daily_task`。
5. 不混入普通未学习新词。

## 2. 队列来源

今日页队列来自 `getReviewStageQueue(data, now)`。

允许进入今日页的词卡：

```text
reviewing
due
overdue
browser_detail + again 后由产品 Adapter 标记为 due 的词卡
```

不得直接进入今日页的词卡：

```text
new
learning
reinforce
downgrade
release
```

说明：

```text
未学习词卡应从单词本场景子页点击“开始学习”进入内联学习。
学习中词卡仍属于场景学习路径，不等于今日待复习。
今日页只处理复习语义。
```

## 3. browser_detail 与 daily_task 边界

`点击单词列表词卡 = 不认识 = browser_detail + again = 加入今日待复习` 是产品规则。

但该规则不得伪装成正式复习。

冻结规则：

```text
点击单词列表产生 review_mode=browser_detail
rating=again
产品 Adapter 把最终 state_after 调整为 due
该词卡进入今日待复习队列
browser_detail 不计入今日已复习
browser_detail 不计入正式复习保持率
```

正式复习只发生在今日页或场景内联学习评分按钮中。

正式复习必须写入：

```text
review_mode=daily_task
rating=again | hard | good | easy
scheduler=ts-fsrs
scheduler_version=5.4.0
fsrs_raw_state_after
state_after
```

## 4. 顶部统计区

今日页顶部固定显示三个统计字段：

```text
待复习
已复习
剩余
```

冻结要求：

```text
字段名不使用“今日待复习 / 今日已复习 / 今日剩余”长文案，避免手机端错行。
数字在上，字段名在下。
三列等宽。
数字使用 tabular-nums。
手机端三位数也不得换行。
```

### 4.1 待复习说明问号

`待复习` 字段右侧可以显示问号图标。

点击问号弹出简短说明：

```text
常规到期词卡按每天复习上限截取；手动点开标记不会的词卡会优先进入今日。
```

冻结要求：

```text
说明放进弹层，不直接显示在统计卡里。
弹层可再次点击问号关闭。
弹层不能挤压统计布局。
```

## 5. 场景筛选

如果今日剩余队列跨多个二级场景，统计区下方显示横向场景筛选 chip：

```text
全部 (N)
二级场景 A (N)
二级场景 B (N)
...
```

规则：

```text
默认选中 全部。
只统计当前剩余队列，不统计已复习词卡。
切换场景后，当前复习卡重置为该场景第一张。
如果只有一个场景或没有剩余词卡，不强行显示筛选条。
```

场景名称必须使用中文显示名。存储 ID 不需要中文化。

## 6. 复习卡片未显示答案状态

复习卡片顶部展示轻量标签：

```text
词频
到期/记忆状态
二级场景
```

卡片正文展示：

```text
问题提示：这个词在真实技术场景中是什么意思？
英文单词
音标 + 读音胶囊
显示答案按钮
```

冻结要求：

```text
未显示答案前不展示来源、下次到期、复习次数等元信息。
这些元信息会干扰即时回忆判断，应从正面卡片删除。
英文单词字号要醒目，但不得大到压迫手机屏幕。
音标胶囊使用浅灰背景，不能过重。
读音按钮必须可点击播放，且不改变记忆状态。
```

## 7. 显示答案后

点击 `显示答案` 后，展示：

```text
中文释义
英文释义
真实场景例句
使用任务
易混词
词族
FSRS 四评分按钮
```

评分按钮必须位于答案详情下方，不能覆盖答案内容。

## 8. FSRS 四评分按钮

四个评分为 FSRS 正式输入：

```text
1 忘记
2 困难
3 良好
4 简单
```

冻结规则：

```text
点击评分后写入 daily_task。
写入后当前词卡从今日剩余队列移除。
页面继续显示下一张待复习词卡。
如果没有剩余词卡，显示完成状态。
```

按钮视觉：

```text
四按钮可使用低饱和背景色区分。
外层不应再叠加厚重轮廓线。
评分区位于内容流中，手机端不固定在底部。
```

决策理由：

```text
手机端固定评分区会占据内容高度，容易遮挡词族、易混词等答案信息。
当前冻结方案采用内容下方评分，保证先看答案，再评分。
```

## 9. 快捷键

PC 端支持键盘辅助：

```text
空格 / Enter：显示答案
1：忘记
2：困难
3：良好
4：简单
```

快捷键不得影响移动端触控体验。

## 10. 空状态

如果当前没有待复习词卡，显示空状态。

分两种语义：

```text
今日待复习已完成：今天曾有待复习，并且已经完成。
暂无待复习词卡：当前没有到期词卡。
```

空状态说明必须表达：

```text
这里不会混入新卡或学习中词卡。
新的到期词卡出现后会自动进入本页。
```

## 11. 与单词本的关系

单词本中的词卡列表点击规则会影响今日页：

```text
用户点击单词列表词卡
-> 视为不认识
-> 写入 browser_detail + again
-> 产品 Adapter 标记为 due
-> 进入今日待复习队列
```

但今日页完成正式复习后：

```text
写入 daily_task
更新 FSRS 状态
从今日剩余中移除
统计页今日已复习增加
```

## 12. 与统计页的关系

今日页影响统计页：

```text
daily_task 计入 今日已复习、Again率、近期保持率、阶段分布、薄弱场景。
browser_detail 只用于把词卡加入今日待复习，不计入正式已复习。
```

不得把浏览写入误算成正式学习成果。

## 13. 与设置页的关系

今日页受设置页学习规则影响：

```text
每天复习上限
弱项巩固开关
逾期优先
复习排序
目标保持率
最大复习间隔
遗忘后重学间隔
```

但设置变化不得立即清空今日页队列；队列应根据最新 `AppData` 和当前时间重新计算。

## 14. 不得回退

以下行为不得重新引入：

```text
今日页混入普通未学习新词。
今日页把学习中词卡当成待复习。
browser_detail 被统计为今日已复习。
顶部字段使用过长文案导致手机错行。
问号说明直接写在统计卡里。
评分按钮固定在底部遮挡答案详情。
点击读音按钮触发评分或写入忘记。
未显示答案前展示来源、下次到期、复习次数等干扰信息。
```

## 15. 回归验收清单

每次改今日页后至少检查：

```text
打开今日页，顶部只显示 待复习 / 已复习 / 剩余。
三位数统计在手机端不换行。
点击待复习问号能看到说明并可关闭。
有多个场景时出现横向场景筛选。
点击显示答案前不出现来源、下次到期、复习次数。
点击显示答案后出现四个评分。
点击四个评分写入 daily_task。
评分后进入下一张卡。
全部完成后显示今日待复习已完成。
单词列表点击写入 browser_detail 后，该词卡进入今日页。
browser_detail 不增加今日已复习。
```

建议命令：

```bash
corepack pnpm run typecheck
corepack pnpm run build
```
