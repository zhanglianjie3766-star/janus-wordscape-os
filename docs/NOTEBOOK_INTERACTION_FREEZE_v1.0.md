# 单词本交互冻结文档 v1.0

本文档冻结“雅努斯词境 OS / TechLex OS”底栏菜单 `单词本` 页的页面结构、场景列表、二级场景子页、学习模式、词卡列表、词卡范围设置和跨页面数据联动。

本文档覆盖单词本主页面及其二级场景子页。未来修改单词本，必须先确认不破坏本文档冻结的学习路径和浏览写入规则。

## 1. 页面定位

单词本是词卡组织、场景进入、学习启动和自然浏览的主入口。

它只回答三个问题：

```text
我有哪些场景词卡？
我要进入哪个二级场景？
我是在当前场景开始学习，还是直接浏览词卡？
```

单词本不是今日复习页，也不是统计页。

## 2. 主页面结构

单词本主页面采用极简结构：

```text
顶部状态统计：未学习 / 学习中 / 待复习
横向领域包入口：全部 / AI编程 / Web3 / 语言运行时 / ...
二级场景卡片列表
```

不得重新加入：

```text
顶部头像区域
日期区域
当前场景说明
学习设置快捷入口
场景列表外层大背景容器
场景卡片内“开始学习”按钮
场景卡片内“单词列表”按钮
粗进度横线
```

## 3. 主页面顶部状态统计

主页面顶部只显示三个状态：

```text
未学习
学习中
待复习
```

当前冻结不在主页面顶部加入 `已掌握`。

原因：

```text
主页面用于判断还要做什么。
未学习、学习中、待复习是行动状态。
已掌握是结果状态，已在二级场景卡片和子页中展示。
主页面加入已掌握会削弱行动优先级，并挤压手机端空间。
```

## 4. 领域包横向入口

横向入口固定为：

```text
全部
各 Domain Pack
```

规则：

```text
默认选中 全部。
点击 全部 后，展示全部范围下聚合的二级场景文件夹。
点击某个 Domain Pack 后，展示该一级领域包下的二级场景文件夹。
每个 chip 显示词卡数量。
横向可滚动，不能挤压换行。
```

## 5. “全部”视图

`全部` 不是平铺单词列表。

冻结规则：

```text
selectedPackId = all
deckMode = folders
activeScene = null
展示所有词卡按 scene_tags 聚合后的二级场景文件夹
```

同一个二级场景跨多个 Domain Pack 时合并为一个场景文件夹。

没有 `scene_tags` 的词卡归入：

```text
未分组
```

## 6. UI 演示数据

当真实词卡数为 0 时，单词本可以进入 UI 演示模式。

冻结规则：

```text
演示数据只用于前端渲染。
不写入 IndexedDB / localStorage。
不生成 ReviewEvent。
不更新 UserMemoryState。
导入真实词卡后，演示数据自动消失。
```

演示模式下：

```text
场景列表可见。
每个场景可进入子页。
学习按钮禁用或提示导入后可用。
词卡详情不可真实写入忘记。
```

## 7. 场景卡片列表

场景卡片是二级场景入口。

卡片结构：

```text
左侧：文件夹图标
中部：二级场景标题
下方：未学习 / 学习中 / 待复习 / 已掌握 统计
右上：掌握百分比 + 右箭头
```

交互冻结：

```text
点击整张场景卡片，都进入二级场景子页。
百分比和箭头只起到“可进入”的视觉暗示。
不得只让百分比或箭头可点击。
```

卡片不再展示：

```text
开始学习按钮
单词列表按钮
粗进度横线
重复场景介绍文字
```

## 8. 场景卡片状态统计

场景卡片展示四个状态：

```text
未学习
学习中
待复习
已掌握
```

映射关系：

```text
未学习 = new 或无 memory state
学习中 = learning / reinforce / downgrade
待复习 = reviewing / due / overdue
已掌握 = release
```

视觉原则：

```text
主色红只用于强调当前行动状态或选中态。
状态文字可以有轻微区分，但不能做成高噪声彩色标签墙。
手机端一行排不下时允许自然换行，但不得挤压标题和箭头。
```

## 9. 二级场景子页入口

进入方式：

```text
点击场景卡片任意区域
```

默认进入：

```text
当前二级场景
状态筛选默认：全部
频率筛选默认：全部
显示开始学习按钮 + 词卡列表
```

不得恢复旧的：

```text
学习模式 / 浏览模式 两个一级大选项
```

原因：

```text
不点击开始学习，直接向下浏览词卡列表，本质就是浏览模式。
点击开始学习，进入内联学习卡片，本质就是学习模式。
不需要额外模式切换。
```

## 10. 二级场景子页顶部

顶部结构：

```text
左：返回箭头圆形按钮
中：当前范围标题
右：设置图标
```

冻结要求：

```text
返回箭头旁不显示“场景列表”文字。
返回箭头旁不显示“全部/IDE界面”等灰色路径文字。
右侧不显示“10张词卡”等重复统计。
右侧只保留设置图标。
中间标题使用当前词卡范围名称，如 IDE界面 / AI编程 / 全部。
标题居中、较粗、较黑、可截断。
```

## 11. 子页状态筛选

状态筛选固定为：

```text
全部
未学习
学习中
待复习
已掌握
```

每个状态必须显示数量。

设计规则：

```text
选中态使用主色红。
未选中态为低噪声灰色文字。
数字与状态文字在同一个 pill 内。
移动端必须保持一行横向滚动，不得强行压缩换行。
```

## 12. 子页范围小字

状态筛选下方显示一行范围说明：

```text
范围：{当前范围} · {当前状态} · {当前词频} · {N} 张
```

冻结要求：

```text
位于状态筛选下方。
左侧与状态筛选第一个按钮文字起点基本对齐。
字体小、灰色、低干扰。
```

## 13. 词卡范围设置

点击子页右上设置图标，打开 `词卡范围` 弹层。

弹层必须自适应手机宽度：

```text
left/right 留安全边距。
max-height 使用视口高度。
内容可纵向滚动。
不得横向溢出屏幕。
```

### 13.1 词频选择

词频选择位于弹层上方。

选项：

```text
全部
F1高
F2中高
F3中
F4低
```

规则：

```text
支持多选。
点击 全部 清空其他词频选择。
其他词频可叠加。
按钮布局为左侧“全部” + 右侧两列词频按钮。
按钮宽度与下方场景按钮协调。
```

### 13.2 场景选择

场景选择位于词频下方。

采用楼梯/级联模式：

```text
左侧：全部 + 一级场景列表
右侧：全部 + 当前一级场景下的二级场景
```

规则：

```text
场景必须单选。
一级和二级必须级联关联。
选择 全部 表示所有一级和二级。
选择某个一级，右侧展示该一级下的二级场景。
选择右侧 全部 表示该一级下全部二级。
选择某个二级，标题联动为该二级场景。
```

显示限制：

```text
一级场景显示字数不超过 5 个字。
二级场景显示字数不超过 7 个字。
超长显示必须截断或使用短中文名。
```

## 14. 开始学习按钮

子页中状态筛选下方保留一个主按钮：

```text
开始学习
开始学习新词
继续学习
开始场景复习
巩固已掌握
```

按钮文案根据当前状态筛选变化。

冻结规则：

```text
点击开始学习不跳转到底栏今日页。
在当前二级场景子页内进入内联学习卡片。
学习范围严格等于当前范围设置 + 当前状态筛选 + 当前词频筛选。
演示数据下禁用或提示导入后可学习。
```

## 15. 内联学习卡片

内联学习卡片与今日页复习卡片保持一致的核心体验。

未显示答案前展示：

```text
词频
记忆状态
场景
当前进度
问题提示
英文单词
音标 + 读音胶囊
显示答案
```

不得展示：

```text
来源
下次到期
复习次数
```

显示答案后展示：

```text
中文释义
英文释义
真实场景例句
FSRS 四评分按钮
```

评分后：

```text
写入 daily_task
更新 FSRS / UserMemoryState
进入当前筛选队列下一张
完成后显示本轮学习完成
```

## 16. 词卡列表

不点击开始学习时，用户自然向下浏览词卡列表。

列表排序：

```text
默认按 data.cards 插入顺序倒序。
后导入 / 后生成的词卡排在前面。
未来如果有 card.created_at，才优先使用 created_at desc。
```

列表项结构：

```text
左侧：读音按钮
中部：英文单词、音标、中文释义摘要
右侧：状态标签、箭头
```

读音按钮必须阻止事件冒泡：

```text
点击读音只播放声音。
不得触发“写入忘记”。
不得展开详情。
```

## 17. 点击词卡列表 = 不认识

这是单词本的核心产品规则。

冻结规则：

```text
无论词卡范围如何选择。
无论状态筛选选择什么。
只要用户点击词卡列表项并展开详情。
就视为不认识该词。
```

系统动作：

```text
写入 review_mode=browser_detail
rating=again
经过产品 Adapter 后 state_after.stage=due
加入今日待复习队列
不伪装成 daily_task
不计入今日已复习
```

## 18. 词卡详情

词卡详情展示：

```text
状态
词频
词性
音标 + 读音
浏览详情写入说明
中文释义
英文释义
真实场景例句
来源
场景
同义词
易混词
词族
```

演示词卡：

```text
不写入 ReviewEvent。
不更新 UserMemoryState。
提示这是 UI 演示词。
```

## 19. 与今日页联动

单词本点击词卡列表会影响今日页：

```text
单词本词卡列表点击
-> browser_detail + again
-> 产品 Adapter 标记 due
-> 今日页待复习增加
```

但今日页正式复习完成后：

```text
daily_task 写入
该词从今日剩余中移除
统计页今日已复习增加
```

PC 端和手机端必须一致。

## 20. 不得回退

以下行为不得重新引入：

```text
全部 tab 直接展示平铺单词列表。
场景卡片里重新放开始学习和单词列表按钮。
点击箭头才能进入子页，点击卡片其他区域无效。
子页恢复学习模式/浏览模式两个大选项。
点击开始学习跳到底栏今日页。
词卡列表点击只展开不写入 browser_detail。
读音按钮触发写入忘记。
browser_detail 被统计为正式已复习。
词卡范围弹层横向溢出手机屏幕。
一级/二级场景显示英文 ID。
```

## 21. 回归验收清单

每次改单词本后至少检查：

```text
单词本默认 全部 显示二级场景文件夹。
场景卡片整张可点击进入子页。
场景卡片显示 未学习 / 学习中 / 待复习 / 已掌握。
子页顶部只有返回、标题、设置图标。
子页状态筛选显示 全部 / 未学习 / 学习中 / 待复习 / 已掌握 和数量。
词卡范围弹层手机端不横向溢出。
词频支持多选，全部可清空。
场景范围为单选级联。
点击开始学习留在当前子页。
内联学习评分写入 daily_task。
点击词卡列表写入 browser_detail + again。
点击读音不写入 browser_detail。
写入 browser_detail 后今日页待复习增加。
browser_detail 不增加今日已复习。
```
